# PRD - مشروع التجارة الإلكترونية (E-Commerce Platform)

**الإصدار:** 1.0
**التاريخ:** يوليو 2026
**الحالة:** مسودة أولى (Draft)

---

## 1. نظرة عامة (Overview)

هذا المستند يوضح متطلبات المنتج (Product Requirements Document) لبناء منصة تجارة إلكترونية متكاملة، تتيح للتجار عرض وبيع منتجاتهم، وللعملاء تصفح المنتجات وشراءها عبر الإنترنت بسهولة وأمان.

### التقنيات المستخدمة (Tech Stack)

| الطبقة | التقنية |
|---|---|
| الباك اند (Backend) | Node.js + Express.js |
| الفرونت اند (Frontend) | Next.js (React) |
| قاعدة البيانات | MongoDB أو PostgreSQL (يُحدد لاحقًا حسب طبيعة البيانات) |
| المصادقة (Auth) | JWT + Refresh Tokens |
| بوابة الدفع | Stripe / Paymob (حسب السوق المستهدف) |
| التخزين السحابي للصور | AWS S3 أو Cloudinary |
| النشر (Deployment) | Docker + CI/CD (GitHub Actions) |

> ملاحظة: تم اختيار Next.js للفرونت اند (وليس Nest.js) بناءً على تأكيدك، حيث إن Nest.js أساسًا فريم ورك للباك اند.

## 2. الهدف من المشروع (Goals & Objectives)

- بناء منصة تجارة إلكترونية سريعة، آمنة، وقابلة للتوسع.
- تمكين العملاء من تصفح المنتجات، البحث، الفلترة، والشراء بسلاسة.
- توفير لوحة تحكم (Admin Dashboard) لإدارة المنتجات، الطلبات، والمستخدمين.
- دعم بوابات دفع متعددة وتجربة Checkout سلسة.
- تحقيق أداء عالٍ (Performance) وتجربة مستخدم ممتازة على الموبايل والديسكتوب.

---

## 3. نطاق المشروع (Scope)

### داخل النطاق (In Scope)
- تسجيل الدخول وإنشاء الحسابات (عملاء + أدمن).
- كتالوج المنتجات (تصنيفات، بحث، فلترة، تفاصيل المنتج).
- سلة الشراء (Cart) وقائمة الأمنيات (Wishlist).
- عملية الدفع (Checkout) وتكامل بوابة دفع.
- إدارة الطلبات (Order Management) للعميل والأدمن.
- تقييمات ومراجعات المنتجات (Reviews & Ratings).
- إشعارات عبر البريد الإلكتروني (تأكيد الطلب، تحديث الحالة).
- لوحة تحكم للأدمن (منتجات، طلبات، مستخدمين، تقارير مبيعات مبدئية).

### خارج النطاق (Out of Scope) - في المرحلة الأولى
- تطبيق موبايل native (iOS/Android).
- نظام تعدد البائعين (Multi-vendor Marketplace).
- الذكاء الاصطناعي في التوصيات (Recommendations Engine).
- الدعم متعدد اللغات والعملات الكامل (يُدرس لاحقًا).

---

## 4. المستخدمون المستهدفون (User Personas)

1. **العميل (Customer):** يتصفح المنتجات، يضيف للسلة، يشتري، يتابع طلباته.
2. **الأدمن (Admin/Store Owner):** يدير المنتجات، المخزون، الطلبات، والمستخدمين.
3. **زائر غير مسجل (Guest):** يمكنه التصفح والشراء كـ Guest Checkout (اختياري).

---

## 5. المتطلبات الوظيفية (Functional Requirements)

### 5.1 المصادقة وإدارة الحسابات
- تسجيل حساب جديد (Email/Password)، وتسجيل دخول عبر Google (اختياري لاحقًا).
- استعادة كلمة المرور (Forgot Password) عبر بريد إلكتروني.
- تحديث بيانات الملف الشخصي والعناوين.
- أدوار مستخدمين: `customer`, `admin`.

### 5.2 كتالوج المنتجات
- عرض المنتجات مع الصور، السعر، الوصف، والتقييم.
- تصنيفات وتصنيفات فرعية (Categories/Subcategories).
- بحث نصي وفلترة (السعر، التصنيف، التقييم، التوفر).
- ترتيب النتائج (الأحدث، الأقل سعرًا، الأعلى تقييمًا).

### 5.3 سلة الشراء والمفضلة
- إضافة/حذف/تعديل الكمية في السلة.
- حفظ السلة للمستخدم المسجل (Persistent Cart).
- قائمة أمنيات (Wishlist).

### 5.4 الدفع والطلبات (Checkout & Orders)
- خطوات الدفع: مراجعة السلة → العنوان → طريقة الدفع → تأكيد الطلب.
- تكامل مع بوابة دفع (Stripe/Paymob) + دعم الدفع عند الاستلام (COD) اختياريًا.
- إنشاء رقم طلب فريد وتتبع حالته (قيد المعالجة، تم الشحن، تم التسليم، ملغي).
- سجل الطلبات السابقة للعميل.

### 5.5 التقييمات والمراجعات
- إضافة تقييم (نجوم) وتعليق نصي بعد الشراء فقط.
- عرض متوسط التقييم لكل منتج.

### 5.6 لوحة تحكم الأدمن
- إدارة المنتجات (إضافة/تعديل/حذف، رفع صور، تحديد المخزون).
- إدارة الطلبات (تغيير الحالة، عرض التفاصيل).
- إدارة المستخدمين (عرض، تعطيل حساب).
- تقارير مبيعات مبسطة (عدد الطلبات، الإيرادات، المنتجات الأكثر مبيعًا).

### 5.7 الإشعارات
- بريد إلكتروني عند: تسجيل حساب جديد، تأكيد الطلب، تغيير حالة الطلب.

## 6. المتطلبات غير الوظيفية (Non-Functional Requirements)

- **الأداء:** زمن استجابة API أقل من 300ms لمعظم الطلبات تحت حمل عادي.
- **الأمان:** تشفير كلمات المرور (bcrypt)، حماية من CSRF/XSS/SQL Injection، استخدام HTTPS.
- **قابلية التوسع:** تصميم الـ Backend بشكل Modular لسهولة إضافة خدمات جديدة (Microservices مستقبلًا).
- **التوافرية:** Uptime مستهدف 99.5%.
- **سهولة الصيانة:** كود منظم حسب مبدأ Separation of Concerns (Controllers, Services, Models).
- **التوافق:** دعم كامل للموبايل (Responsive Design) عبر Next.js.

## 7. البنية التقنية المقترحة (Architecture)

```
[ Next.js Frontend ]  <--- REST API (JSON) --->  [ Express.js Backend ]
                                                        |
                                                  [ Database: MongoDB/PostgreSQL ]
                                                        |
                                            [ Payment Gateway / Cloud Storage ]
```

### هيكلة الباك اند المقترحة (Express.js)
```
/src
  /config        -> إعدادات قاعدة البيانات والمتغيرات البيئية
  /controllers   -> منطق التحكم بالطلبات (Auth, Product, Order...)
  /models        -> نماذج قاعدة البيانات
  /routes        -> تعريف مسارات API
  /middlewares   -> Auth middleware, Error handler, Validation
  /services      -> منطق الأعمال (Business Logic)
  /utils         -> أدوات مساعدة عامة
```

### هيكلة الفرونت اند المقترحة (Next.js)
```
/app أو /pages
  /products
  /cart
  /checkout
  /account
  /admin
/components      -> مكونات UI قابلة لإعادة الاستخدام
/lib              -> استدعاءات API (API client)
/store            -> إدارة الحالة (Zustand/Redux/Context API)
```

## 8. نماذج قاعدة البيانات المبدئية (Data Model Overview)

- **User:** `id, name, email, passwordHash, role, addresses[], createdAt`
- **Product:** `id, name, description, price, images[], category, stock, rating, createdAt`
- **Category:** `id, name, parentCategory`
- **Cart:** `id, userId, items[{productId, quantity}]`
- **Order:** `id, userId, items[], totalPrice, status, shippingAddress, paymentStatus, createdAt`
- **Review:** `id, productId, userId, rating, comment, createdAt`

## 9. أهم نقاط الـ API المقترحة (Sample API Endpoints)

| Method | Endpoint | الوصف |
|---|---|---|
| POST | `/api/auth/register` | تسجيل حساب جديد |
| POST | `/api/auth/login` | تسجيل الدخول |
| GET | `/api/products` | جلب قائمة المنتجات (مع فلترة وبحث) |
| GET | `/api/products/:id` | تفاصيل منتج |
| POST | `/api/cart` | إضافة منتج للسلة |
| GET | `/api/cart` | عرض السلة |
| POST | `/api/orders` | إنشاء طلب جديد |
| GET | `/api/orders/:id` | تفاصيل الطلب |
| GET | `/api/admin/orders` | كل الطلبات (أدمن فقط) |
| PUT | `/api/admin/products/:id` | تعديل منتج (أدمن فقط) |

## 11. مقاييس النجاح (Success Metrics)

- نسبة إتمام عملية الشراء (Checkout Conversion Rate).
- متوسط زمن تحميل الصفحات (< 2 ثانية).
- عدد الطلبات الشهرية.
- معدل الأخطاء في الـ API (< 1%).

## 12. المخاطر والافتراضات (Risks & Assumptions)

- **افتراض:** سيتم تحديد بوابة الدفع النهائية بناءً على السوق المستهدف (محلي/عالمي).
- **خطر:** التعامل مع حالات تزامن المخزون (Stock Concurrency) عند الطلبات المتزامنة يحتاج معالجة دقيقة.
- **افتراض:** قاعدة البيانات ستُحدد نهائيًا (MongoDB مرن للمنتجات، PostgreSQL أفضل للعلاقات المالية الدقيقة).
